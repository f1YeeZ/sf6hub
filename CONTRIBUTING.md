# 贡献指南

感谢你参与 SF6 Training Hub。

## 开始之前

1. 先搜索现有 Issue，避免重复工作。
2. 较大的功能或行为变更请先创建 Issue，说明目标、范围和实现思路。
3. 不要提交密钥、个人数据、上传文件、构建产物或来源不明的第三方素材。

## 开发流程

1. Fork 仓库并从 `main` 创建独立分支。
2. 保持改动聚焦，同时补充或更新相应测试。
3. 中文及其他文本文件统一使用 UTF-8（无 BOM）。
4. 提交前运行：

```bash
npm ci --prefix hub_demo_frontend
npm run verify
```

5. 发起 Pull Request，清楚说明变更目的、验证方式和任何数据库迁移影响。涉及界面变化时可附前后对比图，但不要把临时截图提交到仓库。

## 代码约定

- 遵循现有 Java、JavaScript、Vue 和 CSS 风格，避免无关格式化。
- 数据库结构变更必须新增 Flyway 迁移，不得修改已经发布的迁移文件。
- 新配置项应使用环境变量，并同步更新 `.env.example` 和相关文档。
- API 行为变化应同步更新 `hub_demo_backend/接口文档.md`。
- 依赖升级应保留锁文件，并说明兼容性或安全原因。

提交贡献即表示你有权提供相关内容，并同意按本项目许可证发布你的贡献。
